# 프로젝트 컨텍스트

한국형 생활 계산기 허브 사이드 프로젝트를 진행한다. 목표는 AdSense 광고 수익화이며, 주말 4~5시간 투자, AI 최대 활용이 전제다.

## 사용자 정보

- 6년차 QA, 본업은 LIMS SaaS (Python/Playwright/pytest)
- 사이드 프로젝트는 웹 프론트엔드 영역이며 본업과 분리
- Windows + VS Code 환경, Claude Code Desktop 앱(Code 탭)으로 개발

## 기술 스택(확정)

- Next.js 16 (App Router) + TypeScript
- Tailwind CSS + shadcn/ui
- Cloudflare Pages (배포, 무료 플랜 — 상업적 이용 허용) + GitHub Private Repo
- Google Analytics 4 + Search Console + AdSense
- 계산 로직 단위 테스트: Vitest
- 패키지 매니저: pnpm

## 수익·마케팅 원칙

- 유료 광고(Google Ads, Meta Ads 등) 집행 금지. 광고 수익형 사이트에서 구조적으로 적자
- 백링크 구매 금지(SEO 페널티)
- 마케팅은 SEO + 숏폼(YouTube Shorts·Instagram Reels·TikTok) 유기적 유입에 한정
- 연간 현금 지출은 도메인(약 2만원)만 필수, 그 외는 선택

## 응답 원칙 (최우선)

1. 모든 요청은 먼저 내용을 요약·정리한 뒤 "오케이" 확인을 받은 후에만 진행한다
2. 코드는 승인된 계획이 있을 때만 작성한다. 승인되지 않은 코드 변경은 금지
3. 요청하지 않은 수정·리팩토링을 한 경우 반드시 명시적으로 공개한다
4. 톤: 전문적이고 차분하게. 이모지·과장된 감탄 표현 사용 금지
5. 문서 형식은 한국어 기본, 기술 용어는 원문 병기
6. 표·목록은 정보 비교·열거 목적일 때만 사용. 일반 설명은 문단으로

## 코드 작성 원칙

- 계산기의 세율·공식은 반드시 정부 공식 자료 출처를 주석으로 명시
- 계산 함수는 Vitest 단위 테스트 필수
- Core Web Vitals Good 등급 유지
- 모바일 우선 설계

## AdSense·SEO 원칙

- AI 생성 콘텐츠를 그대로 게시 금지. 본인 검수·실경험 반영 필수
- 각 페이지 고유 콘텐츠 최소 500자
- 필수 페이지(소개/개인정보처리방침/이용약관/연락처) 상시 유지
- 법령·세율 변경 시 페이지 상단에 업데이트 일자 표시

## 문서 포맷 선호

- 품질 검증 표는 Epic(Feature 왼쪽) 포함, Jira Ticket 열 제외, "(Pass/Fail Only)" 라벨 제외
- 차트 선호: 2D + 그라디언트
- 데이터 라벨: "솔루션 별 유형별 현황" 대신 "주요 품질 지표" 사용

## 금지 사항

- 요청하지 않은 기능 추가, 파일 생성, 구조 변경
- 계획 승인 전 코드 작성
- 숫자·세율·계산 공식의 근거 없는 기재
